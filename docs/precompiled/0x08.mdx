## Notes

The point at infinity is encoded with both field `x` and `y` at `0`.

## Inputs

The input must always be a multiple of 6 32-byte values. 0 inputs is valid and returns 1. One set of inputs is defined as follows:

| Byte range | Name |
|-----------:|-----:|
| `[0; 31]` (32 bytes) | x1 |
| `[32; 63]` (32 bytes) | y1 |
| `[64; 95]` (32 bytes) | x2 |
| `[96; 127]` (32 bytes) | y2 |
| `[128; 159]` (32 bytes) | x3 |
| `[160; 191]` (32 bytes) | y3 |

## Output

| Byte range | Name | Description |
|-----------:|-----:|------------:|
| `[0; 31]` (32 bytes) | success | 1 if the pairing was a success, 0 otherwise |

If the input is not valid, all gas provided is consumed and there is no return data, indicating a precompile contract error. Also, if not enough gas was given, there is no return data.
